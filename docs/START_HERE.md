# START HERE

Welcome!

Before writing a single line of code, build a shared understanding of the product.

Complete the following steps in order.

---

# Phase 1 — Understand the Product

## 1. Product Description

Complete:

- `docs/product_description.md`

Define:

- Problem
- Users
- Value Proposition
- MVP

---

## 2. Project Principles

Complete:

- `docs/project_principles.md`

Document the decisions that are unique to this project.

Examples:

- Platforms
- Target devices
- Technical constraints
- UX principles
- AI strategy

---

## 3. Read the Shared Guidance

Read:

- `docs/vibecoding_playbook.md`
- `docs/ux_coaching_mode.md`
- `docs/core_principles.md`
- `docs/ai_working_agreement.md`

These documents describe how the project should be built.

---

# Phase 2 — Validate

Complete:

- `docs/product_discovery_template.md`

Challenge assumptions before implementation.

Focus on:

- Risks
- Unknowns
- Validation
- Small experiments

---

# Phase 3 — Plan

Complete:

- `docs/backlog.md`

Only plan the next few iterations.

Prioritize:

- Learning
- User value
- Fast feedback

---

# Phase 4 — Build

Only begin implementation when:

- Product Description is complete
- Project Principles are defined
- Product Discovery has been reviewed
- The first backlog exists

Build in very small iterations.

---

# During Development

Keep these documents updated:

- Backlog
- Decision Log
- Roadmap
- Changelog
- Project Status

---

# Improving the Template

If you discover improvements to this template:

Document them in:

`template_backlog.md`
