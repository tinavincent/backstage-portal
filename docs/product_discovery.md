# Product Discovery

## Core Assumption

A standard Backstage developer portal provides a stable foundation for cataloging repositories, documentation and technical resources.

The first goal is to validate the vanilla Backstage experience.

Only after that should the portal be extended with project-specific views for vibecoding projects.

## Why Backstage?

Building a custom portal would require solving problems that Backstage already addresses:

- Project catalog
- Documentation (TechDocs)
- Search
- GitHub integration
- Extensible plugin architecture
- Entity relationships

Using Backstage allows the project to focus on product experience rather than platform engineering.

## Unique Value

The unique value of this project is not Backstage itself.

The unique value is creating a product-owner-focused portal for vibecoding projects, AI agents and documentation.

## First Experiment

Validate that Backstage can automatically discover and present multiple vibecoding projects using a shared project structure.

Success criteria:

- Multiple projects appear in the catalog.
- Documentation is available through the portal.
- Navigation between projects feels faster than browsing GitHub repositories manually.
