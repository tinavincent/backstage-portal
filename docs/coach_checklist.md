# Coach Checklist

Use this checklist before implementation.

The goal is to make sure the next step is small, useful and clear.

---

## Ready to Build?

Answer these before starting:

- Do we understand the user problem?
- Is this valuable enough to build now?
- Have we challenged the riskiest assumption?
- Is there a simpler solution?
- Is this the smallest useful version?
- Can we reuse existing UI, data, code or patterns?
- What will the user see or verify afterwards?
- What could break?
- Is the scope small enough for one focused change?

---

## Recommendation

Before implementation, provide:

- Recommended smallest next step
- Expected visible result
- Main tradeoff

Then wait for scope agreement unless implementation was explicitly requested.
